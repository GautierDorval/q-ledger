This release focuses on making Q-Ledger safer to run, easier to reuse, and easier to trust as a public observability surface.

## Highlights

- **Configurable Q-Metrics traceability and disclosure token**
  - Removes hardcoded site references from metric generation.
  - `q_metrics.disclosure_token` is now configurable (default: `Q-LEDGER-DISCLOSED`).

- **Post-publication verification (anti human-error)**
  - Adds a verifier that re-fetches public endpoints and compares canonical hashes vs local outputs.
  - Scripts:
    - `scripts/verify-publication.ps1`
    - `scripts/verify_publication.py`
  - Docs: `docs/post-publication-verification.md`

- **Provider-agnostic input normalization**
  - Defines a minimal `NormalizedRequest` intermediate format (NDJSON/CSV).
  - Adds `scripts/normalize_input.py` adapters (Cloudflare CSV, Nginx combined, AWS ALB, generic JSONL).
  - Adds schema: `schemas/normalized-request.schema.json`
  - Docs: `docs/input-normalization.md`

- **Schemas, tests, CI**
  - JSON Schemas for `q-ledger` and `q-metrics`.
  - Unit tests for critical hashing and parsing helpers.
  - GitHub Actions CI pipeline (Python 3.11).

## Compatibility

- Existing Cloudflare CSV workflows continue to work.
- NDJSON input is supported when:
  - `config.input.format` is `normalized_ndjson`, or
  - the input file ends with `.ndjson` / `.jsonl`.

## Security & privacy

Q-Ledger remains observational:
- no raw log publication
- pseudonymous session inference (salted hash)
- chain-linked ledgers for integrity signals (not a compliance claim)
